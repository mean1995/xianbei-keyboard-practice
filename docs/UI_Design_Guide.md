# UI Design Guide

## 1. 设计目标

RC1.0 面向 9-12 岁儿童。界面应清楚、安静、友好，帮助儿童知道下一步做什么。

核心原则：

- 每个页面只突出一个主要动作。
- 按钮足够大。
- 文案短、具体、温和。
- 错误不惩罚。
- 练习路径始终可返回首页或返回随机练习。
- 不展示复杂统计。

## 2. 首页

首页必须显示：

- 主标题：`仙贝的键盘练习`。
- 欢迎语：`仙贝，今天也来练一会儿吧`。
- 输入法提醒：`练习前，请先关闭中文输入法，切换到英文输入。`
- 入门了解进度。
- 入门练习进度。
- 随机练习完成篇数。
- 入口按钮：入门了解、入门练习、随机练习。
- 清除本地进度。
- 版本号：`版本：RC1.0`。

版本号应弱化显示，不影响儿童主要操作。

## 3. 入门了解界面

入门了解使用屏幕键盘。

设计要求：

- 明确提示用户用鼠标点击屏幕上的键。
- 当前目标键红色高亮。
- 已完成键使用手指颜色。
- 点错不变色，只显示温和提示。
- Shift 组合目标同时高亮两个 Shift 和 baseKey。
- Shift 键缓慢闪烁，不刺眼。

第 27 课先显示大小写讲解页，再进入练习。

## 4. 入门练习界面

入门练习使用实体键盘。

设计要求：

- 先显示“把手放好”准备页。
- 基准位示意区不使用图片，使用 HTML + CSS 显示 A S D F、J K L ; 和 Space。
- 正式练习页提示目标键和手指。
- 鼠标点击屏幕键盘时提示“这一关要按真正的键盘哦”。
- 按错不进入下一题。

大小写课程要明确区分小写和大写输入。

## 5. 随机练习界面

随机练习分为：

- 自由随机练习。
- 短文练习。

自由随机练习显示 4 个入口卡片：

- 基础随机练习。
- 综合随机练习 - 进阶 1。
- 综合随机练习 - 进阶 2。
- 综合随机练习 - 进阶 3。

短文练习只显示 2 个分组卡片：

- Simple English。
- Numbers and Signs。

不要显示内部 30 篇短文卡片。

## 6. 打字练习界面

打字练习不显示屏幕键盘，重点是目标文本和输入框。

设计要求：

- 当前目标文字要大。
- 输入框要大。
- 一次只显示当前组或当前行。
- 正确字符正常显示。
- 错误字符红色显示。
- 完成后提示按 `Enter` 继续。
- 未完成或有错误时，按 `Enter` 不继续。
- 进入下一组或下一行后，输入框自动清空并聚焦。

完成短文后显示：

- 再练一篇。
- 返回随机练习。
- 返回首页。

## 7. 颜色与反馈

- 目标键使用红色高亮。
- 完成键使用手指分区颜色。
- 错误字符使用红色文字或红色背景。
- 错误文案要温和，例如：`这里有一个小错误，改好以后再继续。`
- 鼓励语可以包含：`仙贝，做得很好`、`仙贝，这一行完成了。`

## 8. 不出现的界面

RC1.0 不设计：

- 登录页。
- 用户档案页。
- 排行榜。
- 成就墙。
- 正确率详情页。
- 速度统计页。
- 云同步页。
- 自定义导入文本页。
